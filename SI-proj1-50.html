Representação dos estados¶

Formulação do problema¶

Estado inicial: (x,y,0), em que (x,y) uma clula dentro dos limites do mundo que no obstculo \ Estado final: (x,y,p), em que (x,y) uma clula dentro dos limites do mundo que no obstculo e p >= M o nmero de pontos finais, sendo M o mnimo que deve obter

Supomos que todos os labirintos tm uma moldura quadrada.

  • Operadores de transio entre estados, dado s=(x,y,p)
    • Norte (N): \ Condies: y > 1 e (x,y-1) != obstculo \ Sucessor: s'=copia(s), s[1]=s[1]-1 e s[2]=s[2]+x
    • Sul (S): \ Condies: y < dim-1 e (x,y+1) != obstculo\ Sucessor: s'=copia(s), s[1]=s[1]+1 e s[2]=s[2]+x
    • Este (E): \ Condies: x < dim-1 e (x+1,y) != obstculo \ Sucessor: s'=copia(s), s[0]=s[0]+1 e s[2]=s[2]+x
    • Oeste (O): \ Condies: x > 1 e (x-1,y) != obstculo \ Sucessor: s'=copia(s), s[0]=s[0]-1 e s[2]=s[2]+x
    • Em qualquer um dos operadores:
      • x = 0 se a clula sucessora no tem pastilha
      • x = 1 se a clula sucessora tem a pastilha N
      • x = max(0,5-t) em que t o tempo decorrido at a pastilha ser apanhada se a clula sucessora tem a pastilha D
      • x = t em que t o tempo decorrido at a pastilha ser apanhada se a clula sucessora tem a pastilha C
      • O custo da operao 1 se for a primeira vez que o Pacman visita a clula sucessora e n se for a n-sima

Antes de comearmos a formulao, vamos definir as funes fornecidas no enunciado e construir um conjunto de obstculos.

def exec(p,estado,accoes):
    custo = 0
    for a in accoes:
        seg = p.result(estado,a)
        custo = p.path_cost(custo,estado,a,seg)
        estado = seg
    p.display(estado)
    print('Custo:',custo)
    print('Goal?',p.goal_test(estado))
    return (estado,custo)

def line(x, y, dx, dy, length):
    """Uma linha de clulas de comprimento 'length' comeando em (x, y) na direco (dx, dy)."""
    return {(x + i * dx, y + i * dy) for i in range(length)}

def quadro(x, y, length):
    """Uma moldura quadrada de clulas de comprimento 'length' comeando no topo esquerdo (x, y)."""
    return line(x,y,0,1,length) | line(x+length-1,y,0,1,length) | line(x,y,1,0,length) | line(x,y+length-1,1,0,length)

from searchPlus import *

class PacmanPastilhas(Problem):
    """Obter M pontos (valor passado no parmetro goal) num mundo onde existem pastilhas.
       Um estado  um tuplo ((x,y),p) em que (x,y) representa a posio do Pacman e p  o n de pontos acumulados."""
       
    def __init__(self, initial = ((1,1),0), goal=1, pastilhas={}, obstacles=quadro(0,0,10), dim=10):
        self.initial = initial
        self.pacman = initial[0] # (x,y) que representa a posio inicial do Pacman
        self.pontos = initial[1]
        self.goal = goal # o n mnimo de pontos que o Pacman deve obter
        self.pastilhas = pastilhas # conjunto das pastilhas: dicionrio com chaves {N,D,C} e os valores so uma lista de tuplos correspondentes s posies das pastilhas
        self.obstacles = obstacles # conjunto de obstculos
        self.dim = dim # dimenso do mundo
    
        self.visitadas = [self.pacman]
        self.directions = {"N":(0, -1), "O":(-1, 0),"S":(0, 1), "E":(1,  0)}

        
    def actions(self, state):
        """O Pacman pode mover-se nas quatro direces ortogonais para as quatro clulas
           vizinhas desde que no seja impedido pelos limites do mundo nem por obstculos.
           Uma ao  da forma 'N'. """
        x, y = state[0]
        return [act for act in self.directions.keys() if (x+self.directions[act][0],y+self.directions[act][1]) not in self.obstacles]
        
    def result(self, state, action):
        """Uma ao  da forma 'N' e um estado  um duplo ((x,y),p) em que (x,y) representa a clula onde est o Pacman.
           O resultado ser outro duplo ((x,y),p) em que (x,y) representa a clula onde o Pacman ir estar.
           Caso o Pacman coma uma pastilha, os seus pontos aumentam.
           Tambm  acrescentada a clula sucessora da accao  lista de clulas visitadas."""
        (x,y) = state[0]
        (dx,dy) = self.directions[action]
        result = [(x+dx,y+dy), state[1]]
        for (k,v) in self.pastilhas.items():
            if result[0] in v:
                self.pastilhas[k].remove(result[0])
                if k == "N":
                    result[1] += 1
                elif k == "D":
                    result[1] += max(0, 5-len(self.visitadas)) #len(self.visitadas) = tempo decorrido at a pastilha ser apanhada
                elif k == "C":
                    result[1] += len(self.visitadas)
                    
        self.visitadas.append(result[0])
        return tuple(result)
    
    def goal_test(self, state):
        """Obter pelo menos os pontos passados no parmetro goal."""
        return state[1] >= self.goal
    
    def display(self, state):
        """Imprime um estado."""
        output = ""
        for y in range(self.dim):
            for x in range(self.dim):
                if (x,y) == state[0]:
                    output += "@ "
                elif (x,y) in self.obstacles:
                    output += "= "
                else:
                    output += ". "
                for (k,v) in self.pastilhas.items():
                    if (x,y) in v:
                        output = output[:-2] + k + " "
                if (x+1) % self.dim == 0:
                    output += "\n"
        print(output)
        
    def display_trace(self,d,plan):
        path = list()
        st = self.initial
        for a in plan:
            st = self.result(st,a)
            path.append(st[0])
        
        """ print the state please"""
        output=""
        for j in range(d):
            for i in range(d):
                if self.initial[0] ==(i,j):
                    ch = '@'
                elif (i,j) == path[-1]:
                    ch = "*"
                elif (i,j) in self.obstacles:
                    ch = "="
                elif (i,j) in path:
                    ch = '+'
                else:
                    ch = "."
                output += ch + " "
            output += "\n"
        print(output)
        
    def path_cost(self, c, state1, action, state2):
        """O custo de uma accao  1 se for a 1 vez que visita a clula e n se for a n-sima."""
        return c + self.visitadas.count(state2[0])
    
    def __eq__(self, other):
        """Dois estados so iguais se o Pacman estiver na mesma clula e se os pontos forem iguais."""
        return self.pacman == other.pacman and self.pontos == other.pontos
    
    def __lt__(self,other):
        """Um estado  menor que o outro se os pontos forem menores."""
        return self.pontos < other.pontos

def best_first_tree_search(problem, f):
    f = memoize(f, 'f')
    node = Node(problem.initial)
    if problem.goal_test(node.state):
        return node
    frontier = PriorityQueue(min, f)
    frontier.append(node)
    while frontier:
        node = frontier.pop()
        if problem.goal_test(node.state):
            return node
        for child in node.expand(problem):
                frontier.append(child)
    return None

def uniform_cost_tree_search(problem):
    return best_first_tree_search(problem, lambda node: node.path_cost)

Conclusão¶

Em suma, podemos concluir que, em ambos os mtodos de procura (rvore e grafo), o mtodo que mais facilmente encontra a soluo tima , sem dvida, o custo uniforme. Mas, ainda assim, a procura em grafo com custo uniforme tem um tempo de execuo inferior procura em rvore. Este facto deve-se a que, na procura em grafo, um estado no pode ser expandido mais do que uma vez, mas pode ser revisitado. De uma forma mais detalhada, um estado s revisitado caso o caminho que lhe esteja associado, desde o estado inicial, seja infeior ao caminho, desse mesmo estado, que se encontra na fronteira. Temos ainda que, as solues da procura de custo uniforme em rvore e em grafo so necessariamente as mesmas, facto que pode ser verificado pelos testes realizados mais acima.

Por outro lado, a procura em profundidade aquela em que o tempo de execuo maior. No entanto, aquando da procura em rvore, nem to pouco conseguimos encontrar uma soluo uma vez que o algoritmo entra num ciclo.
